<feature>
  <name>Weekly Stats Types, Goal Resolution, and Aggregation</name>
  <files>src/types/weekly-stats.types.ts, src/types/index.ts, src/utils/weekly-stats.utils.ts, src/utils/__tests__/weekly-stats.utils.test.ts</files>
  <behavior>
    **resolveGoals(weeklyGoals, profileGoals):**
    - Returns weeklyGoals.targetCalories when present, falls back to profileGoals.targetCalories, returns null when neither set
    - Each macro (protein, carbs, fat) resolves independently: weekly > profile > null
    - A goal value of 0 is treated as "set" (not null) — uses ?? not ||
    - When weeklyGoals is null, all values come from profileGoals

    **aggregateWeeklyStats(weekStartDate, mealPlans, goals):**
    - Always returns exactly 7 DayStats entries, Mon through Sun
    - A day with a MealPlan that has 0 dishes AND 0 temporaryMeals → hasData: false
    - A day with no MealPlan document at all → hasData: false
    - A day with at least 1 dish OR 1 temporaryMeal → hasData: true
    - eatenCalories for a logged day = calculateTotalMealPlanNutrition(plan).calories
    - sportCalories for ANY day (logged or not) = calculateTotalBurnedCalories(plan.sports)
    - sportCalories for a missing day (no document) = 0
    - deficit for a logged day with a calorie goal = (targetCalories + sportCalories) - eatenCalories
    - deficit for an unlogged day = null
    - deficit when no calorie goal = null
    - protein/carbs/fat for a logged day = calculateTotalMealPlanNutrition(plan).protein/carbs/fat
    - protein/carbs/fat for an unlogged day = 0
    - totalEatenCalories = sum of eatenCalories for logged days only
    - totalSportCalories = sum of sportCalories across ALL days (logged + unlogged)
    - totalSportSessions = count of all SportActivity objects across ALL days
    - totalDeficit = sum of deficit for logged days (null if no calorie goal)
    - loggedDayCount = count of days with hasData: true
    - avgEatenCalories = totalEatenCalories / loggedDayCount (null if loggedDayCount === 0)
  </behavior>
  <implementation>
    1. Create `src/types/weekly-stats.types.ts` with DayStats, ResolvedGoals, WeeklyStats interfaces (use exact shapes from RESEARCH.md Pattern 2).
    2. Add barrel export to `src/types/index.ts`: `export type { DayStats, WeeklyStats, ResolvedGoals } from "./weekly-stats.types";`
    3. Create `src/utils/weekly-stats.utils.ts` with:
       - `resolveGoals(weeklyGoals: WeeklyNutritionGoals | null, profileGoals: NutritionGoals): ResolvedGoals` — use `??` (nullish coalescing) exclusively, never `||`
       - `aggregateWeeklyStats(weekStartDate: string, mealPlans: MealPlan[], goals: ResolvedGoals): WeeklyStats` — uses `eachDayOfInterval` + `endOfISOWeek` from date-fns to build 7-day skeleton, looks up each day in a Map keyed by date string, calls `calculateTotalMealPlanNutrition` and `calculateTotalBurnedCalories` from `nutrition.utils.ts`, applies isLoggedDay check (dishes + temporaryMeals > 0)
    4. Create `src/utils/__tests__/weekly-stats.utils.test.ts` — test fixtures use plain MealPlan objects (no Firestore). Cover all behavior cases listed above.

    **Important implementation details per user decisions:**
    - Dish.calories is the base value; `calculateTotalMealPlanNutrition` already applies quantity — reuse it, do NOT re-implement
    - Missing protein/carbs/fat on a Dish → treated as 0g (already handled by nutrition.utils)
    - Sport-Sessions = count of individual SportActivity objects (not unique days)
  </implementation>
